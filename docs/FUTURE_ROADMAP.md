# Future Roadmap

The first implementation pass covers Milestone 1 and Milestone 2. Later milestones are listed here so early storage and API decisions do not block planned work.

## Milestone 1: Core graph storage

Create project package, initialize SQLite through GRDB, create core records, create links, and run basic queries.

## Milestone 2: Proposal workflow

Store reviewable proposals, reject proposals, and accept proposals transactionally.

## Milestone 3: Conflict checker

Add persisted conflicts and mechanical conflict detection.

Initial conflicts:

- Invalid reference, blocking
- Possible duplicate object, warning
- Exclusive link overlap, warning
- Manual claim contradiction, warning
- Event time update, warning

Use `ContinuityExclusivityScope` rather than a simple Boolean for relationship exclusivity.

## Milestone 4: Source archive

Copy imported source files into project package, compute SHA-256 checksums, verify sources, and store extracted text generated by app-level parsers.

Do not put PDF parsing, OCR, ebook parsing, audio transcription, video transcription, or web crawling in the engine.

## Milestone 5: Lightweight domain registries

Add registries for object types, relationship types, and claim types.

Registries guide the engine.  They do not turn ContinuityGraph into a rigid ontology system.

## Milestone 6: Import batch contract

Add import batches to group source processing runs and the proposals they generate.

This supports large documents, books, meeting packets, and transcripts without making the proposal queue a loose pile of unrelated items.

## Milestone 7: Query composition

Add useful query helpers for timeline views, graph neighborhoods, claims about records, sources for claims, claims held by an actor, active links at time, basic search, and import batch summaries.

Do not build a custom graph query language yet.

## Milestone 8: Export, import, and sharing foundation

Export selected graph data into `.continuityexport` folders.  Import export packages as proposals.  Preserve IDs when safe, remap on collision, and do not auto-merge duplicate-looking records.

## Milestone 9: Audit history

Add audit entries for creates, updates, archives, proposal acceptance, proposal rejection, conflict status changes, registry changes, and import batch context.

Audit history is not rollback.  Do not implement undo yet.
